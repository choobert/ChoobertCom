Have you ever needed a quick overview of the tables or views in your database? Have you ever wanted to identify which tables/views had a particular column or set of columns?
Fortunately, metadata is saved internally to SQL Server and can be queried in various ways. One of of these ways is through the Information Schema Views.
Via MSDN:
An information schema view is one of several methods SQL Server provides for obtaining metadata. Information schema views provide an internal, system table-independent view of the SQL Server metadata. Information schema views enable applications to work correctly although significant changes have been made to the underlying system tables. The information schema views included in SQL Server comply with the ISO standard definition for the INFORMATION_SCHEMA.
The three views I have used the most are mentioned above, but more are provided and have valuable uses. I highly recommend viewing the MSDN page and checking out all of the available options.
  • TABLES
  • VIEWS
  • COLUMNS
    -- Query to get all tables in a database
    SELECT *
        FROM INFORMATION_SCHEMA.TABLES

    -- Query for all views in a database
    SELECT *
        FROM INFORMATION_SCHEMA.COLUMNS

    -- Query for tables or views with a particular column
    SELECT *
        FROM INFORMATION_SCHEMA.COLUMNS
        WHERE COLUMN_NAME LIKE '%POTENTIAL_COLUMN_NAME%'
For more information view Information Schema Views on MSDN.
